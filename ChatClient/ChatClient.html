Before testing, make sure: Your server is running on your local machine, and the logs show a server on port 8081 and on port 8082 Your server is not set up to allow persistent users. - For the persistent user's data to be secure, clients should not be able to send messages to port 8081 - Instead, the client should ask the server to start a session, and the server will look up the client's chat_id if they're logged in. - The server should send back either an error message or an object containing the chat_id to use, and whether or not the user is resuming a persistent session or creating a new session. Your server is not using SSL. If it is, click here to test it. You can connect new users to the chat using the button in the top right. Note that the send button will send the actual content of the box to the server (after encrypting it). A message must be either a command or a user message Command:[ help | status | format {on|off} ] Recipients:[user1;user2;...] Body:[anything]
