ChatClient browser API

ChatSessionLoader

Asynchronously creates a ChatSession. The ready promise is used to register message handlers when the session becomes available.

ChatSession

Holds the user's conversations, messages, and keysets. Interacts with the server using a websocket connection, and interacts with the application using the provided message handlers.

User messages that are received in real time are passed to the user_message handler. Other user messages are passed to the conversation_cat handler.

Messages

Messages sent with conversationID = 0 are interpreted by the server as commands.

session.enqueue("conversation_request <user_list>", 0)

Request a conversation
user_list is a semicolon-delimited list of usernames

session.enqueue("set_preferences <changed_preferences>", 0)

Update preferences
changed_preferences is a JSON string with the preference keys and new values

These actions should be accessible from the user interface. Other control messages are sent automatically by the ChatSession.
