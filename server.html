ChatServer

Overview

The ChatServer JAR uses port 8081 to handle users creating new chat sessions, and port 8082 to establish WebSocket connections with users. Port 8081 should be firewalled, and a sepearate program on the server should be used to send the join command to it after checking if the user is logged in. Port 8082 should be open to the internet.

Connection Diagram

In order to run the chat app, the server needs to have access to an SSL certificate. This is because the page in which the client runs needs to be loaded over HTTPS in order to use the window.crypto.subtle object.

Configuration

The server reads a configuration file at startup, called config.json. It should be placed in the same directory as the executable, and contain a JSON object.

Configuration options:

{ // defaults to 8081 "authPort": int, // defaults to 8082 "websocketPort": int, // required "keyStoreLocation": absolute path to .jks file -- see below, // defaults to 3306 "mariaDBPort": int, // required "mariaDBUser": string, // required "mariaDBPassword": string, // required "database": MariaDB database name, // defaults to "$cnchat_messages" "tableForMessages": table name, // defaults to "$cnchat_conversations" "tableForConversations": table name, // defaults to "$cnchat_conversations_users" "tableForConversationsUsers": table name }

Note: comments are not allowed in the config file.

Creating a JKS from an SSL keychain

In order to import the SSL certificate at runtime, it must be converted to a Java KeyStore file using the following commands:

  1. openssl pkcs12 -export -in /path/to/cert/fullchain.pem -inkey /path/to/cert/privkey.pem -out tmp.p12 -name <domain_name> This will prompt you for a password for the tmp.p12 keystore.
  2. keytool -importkeystore -deststorepass "nopassword" -destkeypass "nopassword" -destkeystore keyStoreLocation.jks -srckeystore tmp.p12 -srcstoretype PKCS12 -srcstorepass <above_password> -alias <domain_name> The password to open the destination keystore and the password to read the key are both expected to be "nopassword".
